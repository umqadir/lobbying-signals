## What changed and why (context)

This is a lobbying-disclosure dashboard (static site in docs/, SQLite pipeline,
GitHub Actions daily refresh). The Senate LDA API caps page_size at 25 and
supports server-side filing_dt_posted_after filtering. The changes replace
quarter-re-walk ingestion with posted-date-window ingestion:

1. `01_ingest.py`:
   - `ingest_latest()` (called daily by 07_refresh.py) was: re-paginate the two
     most recent completed report quarters (all filing types) + non-original
     types for 4 more trailing quarters. Now: one `ingest_posted_after()` pass
     with a 7-day window, start_year=2020 — everything posted in the last week
     against any report period.
   - New `audit_sample(n, max_minutes)`: random sample of stored is_current=1
     filings re-fetched by UUID from the live API; counts 404s (missing) and
     income mismatches; writes a row to a new `audit_log` table; CLI
     `audit-sample --n --max-minutes`.
   - CLI: `full-sweep` remains (posted-after over ~400 days).
2. `08_trends.py`: `compute_data_checks()` now reads the latest audit_log row,
   exports it as `audit_last` in checks.json, and raises a high-severity flag
   when missing/mismatch counts are nonzero. (Guarded for table-absent.)
3. `.github/workflows/refresh.yml`: daily cron unchanged (10:00 UTC). New
   monthly cron '0 6 3 * *' that runs ONLY the drift audit step
   (`audit-sample --n 40000 --max-minutes 270`) before the normal refresh.
   Weekly/semiannual sweep crons and the one-time backfill input were removed;
   manual `full_sweep` input retained.
4. `README.md`: cadence description updated.
